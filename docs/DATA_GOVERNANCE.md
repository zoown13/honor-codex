# 데이터 수집·검수 정책

## 출처 우선순위

1. 병무청 예우시설 JSONP와 개별 상세
2. 병무청 병역명문가 공지
3. 법제처 자치법규 OPEN API
4. 운영자가 공식 원문을 확인한 수동 입력

각 혜택은 원천 식별자, 원문 URL, 수집시각, content hash, 근거 조항 또는 원문 발췌를 보존합니다.

## 자동 게시

다음 변경만 모든 품질 검사를 통과하면 자동 승인합니다.

- 공식 원천에 새로 등장한 시설
- 전화번호와 홈페이지
- 분류명과 표시 주소의 비핵심 정정
- 동일 행정구역 안에서의 유효한 좌표 정정

## 필수 검수

다음 변경은 소유자가 원문을 확인하고 승인해야 합니다.

- 시설 삭제·종료 또는 행정구역 이동
- 할인율·면제 범위·대상자·증빙·이용 절차·유효기간
- 전국 혜택과 조례의 신규 요약 또는 수정
- 전체 건수 5% 이상 변화 또는 삭제 1% 초과
- 원천 간 내용 충돌, 필수 필드 누락, 잘못된 좌표

원문에 없는 내용은 보충하지 않고 `공식 문서에 명시되지 않음—시설 또는 담당부서 확인 필요`로 표시합니다.

## 최초 기준선 일괄 승인

실데이터 최초 기준선은 다음 조건을 모두 만족할 때만 원천별 일괄 승인할 수 있습니다.

- 같은 원천과 같은 수집시각의 변경이며 원천 접두사, 혜택 유형과 공식 시스템이 서로 일치
- 모든 항목이 `PENDING`, `HIGH`, `ADD`
- `before`가 없고 `after`가 존재
- 한 원천 배치가 2,500건 이하
- 승인 직전 대기 건수와 정렬된 변경 ID의 SHA-256 지문이 검수 화면의 값과 일치

승인은 최대 100건 단위의 조건부 갱신으로 처리하고 멱등 작업 레코드에 원천, 배치, 예상 건수와 지문, 요청자, 시작·완료 시각과 처리 건수를 보존합니다. 변경 레코드에는 작업 ID와 `INITIAL_BASELINE_BULK_APPROVAL` 사유를 기록합니다. 재시도는 같은 요청자와 같은 멱등 키·선택 조건인 경우에만 이어서 처리하며, 다른 작업이나 변경된 선택 집합이 감지되면 중단합니다.

수정·삭제가 섞였거나 원천·수집시각이 혼합된 변경은 일괄 승인하지 않습니다. 승인 작업에는 S3·Amplify·알림 권한이나 호출을 연결하지 않으며 게시 기능은 별도의 성공 게이트가 검증될 때까지 비활성으로 유지합니다.

## 보존과 게시

- 원문 스냅샷: 180일
- Lambda/API 로그: 14일
- 활성·직전 dataset: 항상 보존
- 승인된 변경만 알림 이벤트 생성
- 게시 실패 시 manifest를 바꾸지 않고 직전 dataset 유지

## 사용 조건

병무청 JSONP는 공개 화면에서 호출되지만 정식 버전과 SLA가 명시된 API가 아닙니다. 파일럿은 저빈도 캐시와 출처 표시를 전제로 하며, 공개 확대 전 자동수집과 재사용 조건을 제공기관에 확인해야 합니다.
